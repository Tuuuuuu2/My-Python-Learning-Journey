# 心理学Python代码两则
这是一个包含两个独立的初级Python程序的集合，由心理学专业学生学习python时完成，主要用于心理学相关的娱乐与数据管理。
## 项目概述
本项目包含两个Python程序：
- **心理学灵力检测系统**：娱乐系统，评估研究人员的学术生产力
- **研究团队管理系统**：管理心理学实验人员分配
## 快速开始
## 环境要求
Python 3.6+
无需额外依赖包
## 运行方式
直接运行Python文件：
```python
python "2023341026刘佳.py"
```
## 程序详细介绍
### 1. 心理学灵力检测系统
#### 功能描述
通过分析研究人员在2023-2025年期间的SCI论文发表情况，评估其学术生产力水平。
#### 核心功能
- 数据输入：收集三年的SCI论文发表数
- 质量筛选：确保每年都有论文产出
- 统计分析：计算年均发表量，找出最佳和最差年份
- 等级评定：根据生产力水平给出评级（小牛/牛/牛中牛）
### 2. 研究团队管理系统
#### 功能描述
一个初级的管理系统，用于管理不同心理学实验的研究人员分配。
#### 核心功能
- 添加研究人员：为指定实验添加新的研究人员
- 查看所有人员：显示所有实验及其对应人员
- 查找特定人员：按实验名称查询相关人员
- 退出系统：安全退出程序

### 学习心得与规划
在浏览 GitHub 上其他 Python 项目的过程中，我先看到了别人写的README，当时的第一反应是“怎么别人的README都看起来这么高级？”于是突然心生敬畏……抱着一种“神圣”的感觉开始阅读别人的代码内容。我发现有些人的代码写的很整洁，像经过强迫症式的“仪式化清洗”，有些人的 README 逻辑严密得仿佛是认知心理学教材的示范案例，还有些人的项目结构能让人一眼看到他们的“元认知水平”相当在线。相比之下，我过去的代码更像是自我备忘录式的“自由联想”。这次任务让我意识到，管理项目并不是把文件丢进去那么简单，而是一个系统地呈现思路、组织知识、维持动机的过程——用心理学的话来说，它更像是对“执行功能”的一次综合训练。
完成这次任务的过程也让我第一次意识到，GitHub 仓库本质上是一种“自我呈现（self-presentation）”与“身份建构（identity construction）”。在心理学中，我们经常讨论个体如何通过行为和符号来呈现自己，而在编程世界里，代码、项目结构、commit 信息、README 文档——这些就是构成我“技术身份图式（schema of technical identity）”的关键线索。
我也逐渐明白，一个好的仓库不仅展示“我写过什么”，更展示“我如何思考、如何学习、如何修正错误”。甚至某种程度上，它也是一个观察我自身“元认知监控（metacognitive monitoring）”能力的窗口：我是否能清晰表达自己的意图？是否能让未来的自己理解现在的逻辑？我的计划是否具备持续性和可追踪性？这些都体现在仓库里。
因此，我计划将这个仓库作为我未来学习与成长的长期记录工具：

- 定期更新学习路线图：这会是我“未来导向（future orientation）”和“学习动机调节”的助推器，帮助我持续维持目标导向的行为。
