# 心理学Python代码两则
这个仓库包含两个独立的初级Python程序，由心理学专业学生学习python时完成，主要用于心理学相关的娱乐与数据管理，灵感均来源于日常生活。
## 项目介绍
本项目包含两个Python程序：
- **心理学灵力检测系统**
  - **概述**
     - 该模块通过询问用户近三年在 SCI 期刊上的发表数，构建一个简化的“科研自我评估模型”。程序会自动检验输入是否合法，并根据三年的平均发表量给出一个趣味化的等级评估。
    - 包含初级编程内容：如输入输出、计算、循环等。
    - 适合作为入门程序
  - **核心功能**
	  - 数据输入：收集三年的SCI论文发表数
      - 质量筛选：确保每年都有论文产出
     - 统计分析：计算年均发表量，找出最佳和最差年份
     - 等级评定：根据生产力水平给出评级（小牛/牛/牛中牛）
- **研究团队管理系统**
	- **概述**
		- 这一部分模拟一个非常简化的实验室人员管理界面。		
		- 系统以字典形式保存实验类别与对应研究人员名单，并提供：添加实验人员、查看所有实验与成员、查询特定实验的人员、持续运行的菜单循环
		- 同样也比较适合编程初学者
	- **核心功能**
		- 添加研究人员：为指定实验添加新的研究人员
        - 查看所有人员：显示所有实验及其对应人员
        - 查找特定人员：按实验名称查询相关人员
       - 退出系统：安全退出程序

### 学习心得与规划
在浏览 GitHub 上其他 Python 项目的过程中，我先看到了别人写的README，当时的第一反应是“怎么别人的README都看起来这么高级？”于是突然心生敬畏……抱着一种“神圣”的感觉开始阅读别人的代码内容。我发现有些人的代码写的很整洁，像经过强迫症式的“仪式化清洗”，有些人的 README 逻辑严密得仿佛是认知心理学教材的示范案例，还有些人的项目结构能让人一眼看到他们的“元认知水平”相当在线。相比之下，我过去的代码更像是自我备忘录式的“自由联想”。这次任务让我意识到，管理项目并不是把文件丢进去那么简单，而是一个系统地呈现思路、组织知识、维持动机的过程——用心理学的话来说，它更像是对“执行功能”的一次综合训练。
